# diplom-online-learning
# Эмпирический анализ эффективности алгоритмов Coin Betting

Исходный код к выпускной квалификационной работе «Эмпирический анализ
эффективности алгоритма coin-betting» (Буракова К. В., ЮФУ, кафедра методов
оптимизации и машинного обучения, научный руководитель — проф. Рохлин Д. Б.,
2026).

Репозиторий содержит реализацию parameter-free алгоритмов онлайн-обучения
семейства Coin Betting (COCOB Standard, COCOB Backprop, Reduction to
Magnitude), их сравнение с классическим OGD, с потоковыми библиотеками
River и CapyMOA, а также с онлайн-нейронной сетью (MLP, PyTorch) на задаче
регрессии с функцией потерь Хубера.

## Что делает код

Скрипт `diplom.ipynb` — это полный воспроизводимый пайплайн, который:

1. загружает датасеты California Housing (`T = 20 640`) и Diabetes
   (`T = 442`) из `sklearn.datasets`;
2. реализует четыре алгоритма Coin Betting с нуля (без сторонних
   библиотек) — `OGD`, `COCOB_Standard`, `COCOB_Backprop`,
   `ReductionToMagnitude`;
3. реализует онлайн-обёртку над MLP на PyTorch (`OnlineMLP`) с протоколом
   predict-then-update и `batch_size = 1`;
4. подключает три алгоритма из River и три алгоритма из CapyMOA как
   дополнительные baseline-методы;
5. строит случайные Фурье-признаки (`RBFSampler`) для аппроксимации
   RBF-ядра и прогоняет все 12 методов в едином протоколе на одинаковых
   данных;
6. проводит диагностику знака градиента (проверку гипотезы
   «несимметричной монеты») для COCOB Standard и Reduction to Magnitude
   на линейных признаках;
7. строит все графики и сводные таблицы, использованные в тексте работы.

Файл разбит на 12 пронумерованных блоков:

| № | Блок | Содержимое |
|---|------|------------|
| 1 | Функции потерь | `get_huber_env`, `huber_scalar`, `cumavg` |
| 2 | Алгоритмы Coin Betting | `OGD`, `COCOB_Standard`, `COCOB_Backprop`, `ReductionToMagnitude` |
| 3 | Нейросеть | `OnlineMLP` (PyTorch, Xavier-инициализация, gradient clipping) |
| 4 | Запуск экспериментов | `run_coinbetting`, `run_coinbetting_with_sign_diagnostics`, `run_river`, `run_capymoa`, `run_neural` |
| 5 | Данные | загрузка и предобработка California Housing и Diabetes |
| 6 | Прогоны | последовательный запуск всех методов на обоих датасетах |

> `river` и `capymoa` не обязательны для запуска базовой части эксперимента
> (четыре алгоритма Coin Betting и MLP). Если они не установлены, скрипт
> пропустит соответствующие методы (`RIVER_AVAILABLE` /
> `CAPYMOA_AVAILABLE` выводятся в консоль при запуске) и продолжит работу
> без них — упадёт только построение итоговых графиков, где сравниваются
> все 12 методов.
>
> `capymoa` требует установленной Java (JDK ≥ 11), так как использует
> библиотеку MOA через интерфейс JPype.

## Запуск

```bash
python experiment.py
```

Время выполнения на полном датасете California Housing
(`T = 20 640`, 12 методов) — порядка 10–20 минут на обычном ноутбуке без
GPU; основное время уходит на онлайн-обучение MLP и на алгоритмы CapyMOA.

Скрипт выводит в консоль:

- статус доступности River / CapyMOA / PyTorch;
- ход выполнения по этапам (`[1/6] ...`, `[2/6] ...` и т.д.);
- результаты диагностики знака градиента (доли `sign(g)=+1/-1`, доля
  повторов, скользящая доля доминирующего знака);
- три сводные таблицы с финальными значениями накопленной средней потери
  $L_{\mathrm{avg}}$ для линейного режима, Фурье-режима на California
  Housing и Фурье-режима на Diabetes.

Графики сохраняются в текущую директорию (или в `figures/`, если вы
поменяете пути в коде) и одновременно показываются через `plt.show()`.

## Воспроизводимость

Все случайные операции зафиксированы единым `seed = 42`:

- `np.random.seed(42)` — перемешивание (`shuffle`) обоих датасетов перед
  запуском потоковой симуляции;
- `RBFSampler(..., random_state=42)` — генерация случайных частот для
  Фурье-признаков (одна и та же выборка частот для всех 12 методов);
- инициализация весов MLP идёт через тот же глобальный seed NumPy/PyTorch.

Порядок подачи объектов `(x_t, y_t)` идентичен для всех методов на одном
датасете — это гарантирует, что сравнение проводится в равных условиях.

- График проверки гипотезы «несимметричной монеты»
  (`fig_sign_diagnostic.png`, рис. в разделе 6.1 ВКР) — блок 7–8 кода.

  `print_table(..., last_n=50)`.

## Известные ограничения реализации

- Нормировка признаков и целевой переменной (`StandardScaler`)
  выполняется в пакетном режиме на всём датасете до начала симуляции
  потока — это отступление от строгой онлайн-постановки (подробное
  обсуждение — в разделе «Наборы данных» текста работы).
- Для CapyMOA каждая модель проходит поток данных независимо (заново
  создаётся `NumpyStream`), так как один и тот же поток нельзя
  «перемотать» для нескольких моделей подряд.
- Диагностика знака градиента запускается только для COCOB Standard и
  Reduction to Magnitude на линейных признаках California Housing —
  именно эти два метода расходятся в данном режиме.
